# Wind Turbine SCADA Analysis: Prediction and Forecasting

This repository contains a complete exploratory and modelling analysis of a wind turbine SCADA dataset. The project studies the behaviour of active power generation, evaluates the theoretical power curve as a physics-based benchmark, and develops machine learning models for both active power prediction and short-term forecasting.

The main analysis is implemented in a Jupyter Notebook located in `src/main.ipynb`.

---

## Project objective

The objective of this project is to analyse wind turbine SCADA data and build interpretable machine learning models for two related tasks:

1. **Active power prediction**  
   Estimate the turbine active power at the current timestamp using meteorological, temporal and engineered operational features.

2. **Short-term active power forecasting**  
   Forecast active power at two future horizons:
   - **10 minutes ahead**
   - **1 hour ahead**

The project also compares machine learning models against strong physical and temporal baselines, rather than evaluating them in isolation.

## Dataset

The dataset used in this project is the public **Wind Turbine SCADA Dataset** available on Kaggle.

The original file used in the notebook is:

```text
data/T1.csv
```

The dataset contains 10-minute SCADA observations from a wind turbine. The main variables are:

| Original variable | Renamed variable | Description |
|---|---|---|
| `Date/Time` | `date` | Timestamp of the observation |
| `LV ActivePower (kW)` | `active_power_kW` | Measured active power generated by the turbine |
| `Wind Speed (m/s)` | `wind_speed_mps` | Wind speed measured in m/s |
| `Theoretical_Power_Curve (KWh)` | `theoretical_power_kW` | Theoretical power curve used as a physics-based reference |
| `Wind Direction (°)` | `wind_direction_deg` | Wind direction in degrees |

Although the theoretical power curve is originally labelled in kWh, it is treated as a power reference in this analysis because it represents the expected turbine power output under the corresponding wind conditions.

## Main workflow

The notebook is structured into the following main sections:

1. **Data loading and initial inspection**
   - Load the raw SCADA dataset.
   - Inspect shape, columns, missing values and data types.
   - Rename variables for readability.

2. **Basic preprocessing and variable transformation**
   - Convert timestamps into datetime format.
   - Transform wind direction into sine and cosine components.
   - Create interpretable time-related variables.

3. **Exploratory Data Analysis**
   - Analyse distributions of active power, wind speed, theoretical power and wind direction.
   - Study outliers and saturation effects.
   - Explore the relationship between wind speed and generated power.

4. **Physics-based benchmark**
   - Evaluate the theoretical power curve as a baseline model.
   - Use this benchmark to assess whether machine learning models add value.

5. **Temporal behaviour and autocorrelation**
   - Analyse daily patterns and temporal persistence.
   - Study autocorrelation in active power and theoretical power.
   - Assess the relevance of lagged values for forecasting.

6. **Calendar-based feature exploration**
   - Analyse possible differences by hour, weekday, weekend and month.
   - Create cyclic time features.

7. **Data quality analysis**
   - Inspect negative active power observations.
   - Analyse time gaps between consecutive observations.
   - Create gap-related indicators.

8. **Prediction modelling**
   - Use residual modelling to correct the theoretical power curve.
   - Compare Ridge, ElasticNet, Spline Ridge, Random Forest and HistGradientBoosting.
   - Explore a two-stage model for non-generation events.

9. **Forecasting modelling**
   - Build a multi-horizon forecasting dataset.
   - Create lagged and rolling features.
   - Forecast active power 10 minutes and 1 hour ahead.
   - Compare models against a persistence baseline.

10. **Global conclusions and limitations**
    - Summarise the main modelling findings.
    - Discuss the limitations of the available SCADA variables.
    - Propose possible future improvements.

## Feature engineering

Several feature engineering techniques are applied throughout the project.

### Wind direction transformation

Wind direction is circular, so the raw degree value is transformed into sine and cosine components:

```python
wind_direction_x = cos(wind_direction_deg)
wind_direction_y = sin(wind_direction_deg)
```

This avoids artificial discontinuities between directions close to 0° and 360°.

### Cyclic time features

Hour, month and day-of-week variables are transformed into sine and cosine features to represent their cyclic nature:

```python
hour_sin, hour_cos
month_sin, month_cos
dow_sin, dow_cos
```

### Gap-related features

Two features are created to represent irregular time gaps:

```python
gap_minutes
after_gap
```

These variables help the models identify observations that occur after missing periods or possible turbine stops.

### Forecasting lag and rolling features

For the forecasting task, lagged and rolling features are created for active power and key explanatory variables, including:

```text
active_power_kW_lag_1
active_power_kW_lag_6
active_power_kW_lag_12
active_power_kW_roll_mean_6
active_power_kW_roll_std_6
wind_speed_mps_lag_6
theoretical_power_kW_roll_mean_12
```

These variables provide information about the recent behaviour of the turbine and the surrounding wind conditions.

## Prediction task

### Problem definition

The prediction task estimates the active power at the same timestamp:

```text
X(t) → active_power_kW(t)
```

Since the theoretical power curve already provides a strong physics-based estimate, the main modelling strategy is not to replace it, but to correct its residual error:

```text
residual = active_power_kW - theoretical_power_kW
```

The final prediction is reconstructed as:

```text
active_power_prediction = theoretical_power_kW + predicted_residual
```

### Prediction models tested

The following models are evaluated:

- Theoretical power curve benchmark
- Ridge Regression residual model
- ElasticNet residual model
- Spline Ridge residual model
- Random Forest residual model
- HistGradientBoosting residual model
- Two-stage prediction model for non-generation events

### Prediction results

| Model | RMSE | R² |
|---|---:|---:|
| Ridge residual | 408.07 | 0.9075 |
| ElasticNet residual | 424.78 | 0.8998 |
| Theoretical curve | 470.71 | 0.8770 |
| Spline Ridge residual | 518.07 | 0.8510 |
| Random Forest residual | 829.70 | 0.6178 |
| HistGradientBoosting residual | 869.65 | 0.5801 |

### Prediction conclusion

The best prediction model is the **Ridge residual model**. It improves the theoretical power curve benchmark while remaining simple, stable and interpretable.

More complex nonlinear models such as Random Forest and HistGradientBoosting perform worse, suggesting that the residual component does not contain a strong nonlinear structure that can be reliably learned from the available variables.

## Non-generation analysis

A two-stage model is also tested to address observations where the turbine does not generate power despite having non-zero theoretical power.

The two-stage approach consists of:

1. A classifier that predicts whether the turbine is generating energy.
2. A regression model that predicts active power only when the turbine is predicted to be generating.
3. A forced prediction of zero active power when the classifier predicts a non-generating state.

The tested classifiers are:

- Logistic Regression
- Random Forest Classifier

### Classification results

| Model | Accuracy | Precision | Recall | F1 |
|---|---:|---:|---:|---:|
| Logistic Regression | 0.9004 | 0.9542 | 0.9176 | 0.9356 |
| Random Forest | 0.9132 | 0.9192 | 0.9756 | 0.9466 |

Although the classification models achieve reasonable performance, the complete two-stage regression strategy does not improve the final active power prediction:

| Model | RMSE | R² |
|---|---:|---:|
| Two-stage model | 489.19 | 0.8671 |

Therefore, the two-stage model is analysed as an additional experiment but is not selected as the final prediction model.

## Forecasting task

### Problem definition

The forecasting task estimates future active power using only information available at the current timestamp:

```text
X(t) → [active_power_kW(t + 10 min), active_power_kW(t + 1 h)]
```

This is formulated as a **multi-output regression problem** with two targets:

```text
target_10min
target_1h
```

### Forecasting baseline

A persistence baseline is used as the main benchmark:

```text
future active power = current active power
```

This is a strong baseline for short-term forecasting because turbine active power shows strong temporal persistence, especially at the 10-minute horizon.

### Forecasting models tested

The following models are evaluated:

- Persistence baseline
- Ridge multi-output regression
- ElasticNet multi-output regression
- Random Forest multi-output regression
- HistGradientBoosting multi-output regression

### Forecasting results

| Model | Horizon | RMSE | R² |
|---|---|---:|---:|
| Ridge multi-output | 10 min | 226.21 | 0.9733 |
| ElasticNet multi-output | 10 min | 226.27 | 0.9733 |
| Persistence baseline | 10 min | 226.74 | 0.9731 |
| Random Forest multi-output | 10 min | 228.59 | 0.9727 |
| HistGradientBoosting multi-output | 10 min | 230.66 | 0.9722 |
| Ridge multi-output | 1 h | 478.35 | 0.8805 |
| ElasticNet multi-output | 1 h | 478.38 | 0.8804 |
| Persistence baseline | 1 h | 485.59 | 0.8768 |
| Random Forest multi-output | 1 h | 501.44 | 0.8686 |
| HistGradientBoosting multi-output | 1 h | 504.64 | 0.8670 |

### Forecasting conclusion

The forecasting results show that simple regularized linear models are the only models that slightly improve the persistence baseline.

The improvement is small, especially for the 10-minute horizon, which indicates that the problem is strongly dominated by temporal persistence. The 1-hour horizon provides slightly more room for improvement, but the gain over the persistence baseline remains moderate.

The final forecasting model selected is **Ridge multi-output regression**, due to its simplicity, stability and interpretability.

---

## Main conclusions

The project shows that the theoretical power curve is a very strong physics-based benchmark for wind turbine active power prediction.

For the prediction task, the best strategy is to correct the theoretical power curve through a Ridge residual model, rather than replacing the physical benchmark with a highly flexible machine learning model.

For the forecasting task, the current active power is already a very strong predictor of future power, especially 10 minutes ahead. Ridge and ElasticNet slightly improve the persistence baseline, but complex nonlinear models do not provide additional benefits.

Overall, the final selected models are:

| Task | Final model |
|---|---|
| Active power prediction | Ridge residual correction |
| Short-term forecasting | Ridge multi-output regression |

## Limitations

The main limitation of this project is the lack of detailed turbine operational information in the public Kaggle dataset.

The dataset does not include variables such as:

- Turbine status
- Alarm logs
- Maintenance records
- Curtailment indicators
- Grid connection status
- Pitch angle
- Rotor speed
- Generator speed
- Nacelle position
- Availability signals

This limitation is especially important for observations where the theoretical power is high but the actual active power is close to zero. These situations may be caused by shutdowns, faults, curtailment, maintenance operations or grid restrictions. Since these operational states are not directly observed, the models can only infer them indirectly from meteorological, temporal and lagged variables.

Another limitation is that the dataset contains observations from a single turbine and one year of operation. Therefore, the conclusions may not generalize directly to other turbines, years, wind farms or geographical locations.

For the forecasting task, the models are also limited by the absence of future meteorological forecasts. In a real operational environment, short-term wind forecasts and richer SCADA variables would likely improve both 10-minute and 1-hour ahead predictions.

## Future work

Future improvements could include:

- Adding richer SCADA variables such as pitch angle, rotor speed, generator speed and turbine status.
- Incorporating alarm logs, maintenance events and curtailment indicators.
- Using external short-term weather forecasts for the forecasting task.
- Testing the methodology on several turbines and multiple years.
- Building a dedicated classification model for turbine availability or operational state.
- Comparing direct forecasting with recursive and sequence-based forecasting strategies.
- Evaluating probabilistic forecasts or prediction intervals instead of only point predictions.

## Reproducibility notes

The notebook uses chronological train/test splits for prediction and forecasting to avoid temporal leakage.

Several models use `random_state=42` to make the results reproducible where possible. However, exact results may vary slightly depending on package versions, operating system and hardware.

The `.venv/` directory should not be committed to version control. It is recommended to include a `.gitignore` file with at least:

